# Feedback Taxonomy

## Feedback Categories

- Accepted
- Edited before use
- Rejected
- Escalated to human owner
- No approved source found

## Rejection Reasons

- Answer unclear
- Source missing or weak
- Policy-sensitive
- Low confidence
- Wrong workflow moment
- Needed human judgment

## Why This Matters

Feedback should help the product team understand whether the issue is model quality, source quality, workflow fit, user trust, or governance design.
