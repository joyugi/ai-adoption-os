# Rollout Plan

## Phase 1: Discovery

Goal: Identify one workflow where an AI assistant could reduce search time and improve employee experience.

Activities:

- interview target users
- identify top recurring questions
- audit existing knowledge sources
- classify risk levels
- define success metrics

## Phase 2: Limited Pilot

Goal: Test whether users adopt the assistant in a real workflow.

Pilot scope:

- one team
- one workflow
- approved knowledge base
- clear human escalation owner
- limited set of question categories

Pilot success criteria:

- repeat usage by target users
- useful answer acceptance rate
- low unsafe-use incidents
- clear source coverage gaps
- positive time-saved signal

## Phase 3: Instrumentation and Learning

Goal: Understand whether adoption is real and where trust fails.

Track:

- accepted responses
- edited responses
- rejected responses
- escalated questions
- unresolved questions
- helpfulness rating
- estimated time saved
- risk category

## Phase 4: Controlled Expansion

Goal: Expand only after the pilot demonstrates sustained usage, trust, and business value.

Expansion criteria:

- improved source quality
- clear escalation patterns
- acceptable risk profile
- stakeholder sponsorship
- measurable productivity gain
- user pull from adjacent teams

## Phase 5: Production Governance

Goal: Operate the assistant as a governed AI product.

Ongoing operating model:

- source ownership
- feedback review cadence
- risk review
- adoption reporting
- model and content quality monitoring
- governance updates
